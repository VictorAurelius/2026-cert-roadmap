Insight nhanh

📚 Oracle SQL dày đặc nhất theo từng bảng — 22 từ (27% tổng), xoay quanh constraint, function & transaction. May mắn là học sớm ngay giai đoạn tạo đà (T1–3).

☕ Java nặng nhất khi gộp theo cert — 26 từ (32%) = 17 Core + 9 Collections/Streams. Khớp đúng việc Java là chặng khó & dài nhất (T4–9).

🔁 Vài khái niệm lặp xuyên suốt 3 cert — transaction, propagation, scope, constraint. Học một lần, gặp lại ở cả SQL, Java lẫn Spring → lãi kép.

⏱️ 5 từ/ngày phủ hết 81 từ trong ~16 ngày — nhưng nên rải đều theo lý thuyết suốt 13 tháng để nhớ lâu, không nhồi một lần.

Cách đếm
Mỗi dòng trong bảng = 1 mục từ. Các cụm đôi (vd. heap / stack, autoboxing / unboxing) tính là 1 mục theo đúng cách trình bày của plan. Tổng: 15 + 17 + 9 + 22 + 18 = 81.
Gộp theo chứng chỉ
Java = Core (17) + Collections/Streams (9) = 26. "Chung" giữ riêng vì là từ học thuật nền, dùng chéo cả 3 cert.
Mốc thời gian biểu đồ tích lũy
Ánh xạ theo thứ tự ôn của repo: SQL (T1–3) → Java (T4–9) → Spring (T10–13); từ vựng "Chung" học ngay tuần đầu. Đường tích lũy minh họa nhịp, không phải lịch cứng.
